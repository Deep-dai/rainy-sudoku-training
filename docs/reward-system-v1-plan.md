# Rainy 数独贴纸奖励系统 V1 开发计划

- 版本：V1.7
- 建立日期：2026-07-12
- 当前状态：V1 开发完成
- 本期范围：奖励阶梯、贴纸发放、重复贴纸升级、贴纸收藏页
- 暂不包含：阶段奖励、连续签到、兑换货币、多人档案

## 1. 产品目标

孩子正确完成数独后立即获得一张与题目难度匹配的电子贴纸。奖励重点放在完成和独立思考，不以速度或首次提交是否正确作为扣减条件。

## 2. 奖励阶梯

第一版包含 20 张贴纸，每档 4 张。

| 档次 | 适用题目 | 贴纸主题 |
| --- | --- | --- |
| 1级 小贴纸 | 6阶超级简单 | 星星、彩虹、草莓、小雨伞 |
| 2级 可爱动物 | 6阶非常简单 | 小兔、小猫、小鸭、企鹅 |
| 3级 闪亮动物 | 6阶简单、9阶超级简单 | 狐狸、小熊猫、海豚、猫头鹰 |
| 4级 珍藏毛绒 | 6阶高手、9阶非常简单（80%） | 粉色兔兔、灰色兔兔、毛绒长颈鹿、星光伊布 |
| 5级 超级珍藏 | 9阶非常简单（20%）、9阶高手、9阶超级高手 | 金瞳小黑龙、星雪小白虎、皇冠小黄鸭、月光光煞 |

除 9 阶非常简单采用 4/5 级概率奖励外，其余难度固定对应一个奖励档次，不会随机降档。

## 3. 贴纸发放规则

1. 题目全部正确并通过“交作业”或“交卷”后发放。
2. 练习模式提交错误后可以继续修改，最终正确时正常发放。
3. 比赛模式只有正确交卷才发放，错误交卷和时间到不发放。
4. 使用提示不降低贴纸等级；全程未使用提示时显示“独立完成”皇冠。
5. 完成用时不影响奖励。
6. 中途更换新题不发放奖励。
7. 同一道题在当前游戏状态中只能领取一次。

## 4. 抽取和重复规则

- 当前档次存在未收集贴纸时，80%优先抽取未收集贴纸，20%从全档随机抽取。
- 当前档次全部收齐后，从该档全部贴纸中随机抽取。
- 第 1 次获得：解锁普通贴纸。
- 第 2 次获得：升级为一星微光。
- 第 3 次获得：升级为二星闪亮边框。
- 第 4 次获得：升级为三星彩虹闪卡。
- 第 5 次及以后：保持三星，继续累计拥有数量。

界面使用“贴纸变得更闪亮了”描述重复奖励，不使用负面的“重复”提示。

## 5. 界面范围

### 完成揭晓

- 正确完成后显示完成用时、贴纸礼物和揭晓动画。
- 展示贴纸名称、等级、星级变化和新收集/升级文案。
- 无提示完成时展示“独立完成”皇冠。
- 提供“查看贴纸”和“再来一题”操作。

### 我的贴纸

- 顶部入口显示已解锁数量，例如 `7/20`。
- 收藏页按五个等级分区展示。
- 未获得贴纸显示剪影，已获得贴纸显示名称、星级和累计数量。
- 点击已获得贴纸可以打开大图，查看完整插画、等级、星级和累计数量。
- 手机采用 2 列，iPad 采用 4 列，较宽屏幕采用 5 列。
- 动画尊重系统的“减少动态效果”设置。

## 6. 技术方案

- 新增 `scripts/app-rewards.js`，集中管理贴纸目录、难度映射、抽取、升级、渲染和存档。
- 使用独立本地存储键 `rainy-sudoku-rewards-v1`，不改写现有设置存档。
- 游戏状态增加 `usedHint`、`rewardGranted` 和 `currentReward`。
- 正确提交后调用奖励模块；错误提交、时间到和中途换题不调用。
- 贴纸使用本地离线视觉资源和 CSS 闪卡效果，纳入 PWA 缓存。

## 7. 验收标准

- [x] 8 种棋盘和难度组合均映射到正确奖励档次。
- [x] 练习模式错误后修正仍能获得奖励。
- [x] 比赛模式正确获得奖励，错误和超时不获得奖励。
- [x] 使用提示仍有贴纸，但不显示独立完成皇冠。
- [x] 同一道题不能重复领取。
- [x] 贴纸首次解锁和第 2 至第 4 次升级正确。
- [x] 刷新页面后收藏和数量不丢失。
- [x] 损坏或旧版存档不会阻止应用启动。
- [x] 手机和常见 iPad 横竖屏无横向溢出，操作区域可触控。
- [x] 离线缓存包含奖励脚本与视觉资源。
